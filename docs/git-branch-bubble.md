# Git 分支提示（仅「仓库变化」时气泡一句）

> 分支提示**不再**跟「心情台词」绑在一起推前端，也**不会**按小猪轮询周期刷分支名。  
> 设计目标：**只有工作区 git 状态相对上次快照发生变化**时，才用气泡说一句提示（切换分支、检出到新分支、同分支上新提交等）。

---

## 1. 触发条件（谁驱动）

| 项目 | 说明 |
|------|------|
| **数据源** | 小猪进程对工作区路径执行 `read_git_state`：读 `.git/HEAD` 得到**分支名**（或 detached 短 hash），并执行 `git rev-parse --short HEAD` 得到 **HEAD 短 SHA**。 |
| **轮询** | 与 Cursor/进程探测同周期，约 **每 3 秒**（每 20 个 tick × 150ms）读一次磁盘上的 git 状态。 |
| **提示触发** | 将当前 `(branch, head_short)` 与**内存中上一快照**比较：**仅当二者任一与上次不同**时，才可能生成一句 `GitTip` 文案并送到 WebView。 |
| **首次启动** | 第一次读到快照只**播种**，**不弹气泡**（避免一打开就念分支）。 |

> 说明：这不是在 shell 里 hook `git commit` 命令，而是**观察 `.git` 结果**；一般提交后几秒内 HEAD 或 ref 会变，短 SHA 会变，从而触发「新提交」提示。若要 **0 延迟** 或 **仅特定子命令**，可后续加 `git config core.hooksPath` + 写 `~/.nixie/git-pulse.json` 或在本进程对 `.git` 做 `notify` 监听。

---

## 2. 提示文案规则（`main.rs` 内 `build_git_tip_message`）

| 变化 | 示例气泡 |
|------|----------|
| 分支名变化（且当前有分支名） | `⏣ 已切换到 feature/foo`（过长会截断加 `…`） |
| 从「无分支信息」到有 | `⏣ main` |
| 从有分支到读不到（如删掉 `.git`） | `⏣ 无 git 分支信息` |
| **分支名相同**且 **短 SHA 变化** | `⏣ main 新提交 · a1b2c3` |

`dirty`（未提交文件数）**不参与**提示，避免保存一下文件就刷屏。

---

## 3. 与心情台词的关系

- **`MoodChanged` → `updateMood(mood, label, hasExt, quote)`**  
  只负责皮肤 class、随机台词、Hook 小圆点；**不再**带分支字符串。
- **`NativeHintsChanged` → `syncNativeHints(hasExt)`**  
  约每 9 秒仅在**心情未变**时同步小圆点；**不**更新分支。
- **`GitTip` → `showGitTip(message)`**  
  专用气泡，约 **2.8s**；与 `showToast` 一样会占 `#mood-text`，但**不**跳 `pet-jump`。

若同一轮里既换心情又有 git 变化，为避免盖住心情台词，会**跳过**本次 `GitTip`（快照仍会更新，避免重复排队；极端同时发生概率低）。

---

## 4. 隐私

仍仅读**本地** `.git` 与 `git` 子进程，**无网络**。
